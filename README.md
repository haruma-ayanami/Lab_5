Лабораторно-практическая работа №5

Структурные паттерны проектирования: Компоновщик, Адаптер и Фасад

Выполнил: Ткачев А. Д.
Группа: 2407СА1
Проверил: Макаров М. С.

Цель работы

Изучить структурные паттерны проектирования «Банды четырёх (GoF)» на примере разработки приложения, моделирующего работу файлового менеджера с поддержкой различных файловых систем и облачных хранилищ.
Освоить на практике паттерны Компоновщик (Composite) для представления иерархической структуры файлов и папок, Адаптер (Adapter) для унификации доступа к различным файловым системам и Фасад (Facade) для предоставления простого интерфейса выполнения типовых операций, таких как синхронизация и резервное копирование.

Задание
Разработать консольное приложение, моделирующее работу файлового менеджера.
Программа должна позволять:
Создавать иерархическую структуру файловой системы, состоящую из файлов и папок. Для этого необходимо использовать паттерн Composite, в котором файлы выступают в роли листовых элементов структуры, а папки являются составными объектами, способными содержать другие элементы файловой системы. Реализовать операции рекурсивного обхода структуры файловой системы, включая вычисление общего размера папок, удаление элементов и копирование файлов. Интегрировать различные файловые системы через единый интерфейс с использованием паттерна Adapter. Адаптер должен предоставлять общий интерфейс для основных операций работы с файлами, таких как просмотр содержимого директории, чтение файла, запись данных и удаление элементов. Предоставить упрощённый интерфейс для выполнения типовых операций с использованием паттерна Facade. В качестве примеров операций можно реализовать синхронизацию локальной папки с облачным хранилищем и создание резервной копии файлов.

Теоретическое обоснование
Структурные паттерны проектирования используются для организации взаимодействия между классами и объектами системы. Они позволяют упростить структуру программы и обеспечить более удобное взаимодействие между компонентами системы.
Паттерн Компоновщик (Composite) предназначен для представления иерархических структур объектов в виде дерева. Он позволяет клиентскому коду одинаково работать как с отдельными объектами, так и с их группами. В контексте файловой системы этот паттерн позволяет представить папки и файлы как элементы одной структуры. Папки могут содержать другие элементы, тогда как файлы являются конечными объектами структуры.
Паттерн Адаптер (Adapter) используется для обеспечения совместимости между интерфейсами различных классов. Он преобразует интерфейс одного класса в интерфейс, ожидаемый клиентским кодом. Благодаря этому становится возможной работа с различными файловыми системами через единый интерфейс.
Паттерн Фасад (Facade) предоставляет упрощённый интерфейс для сложной подсистемы. Он скрывает внутреннюю логику работы нескольких классов и предоставляет клиенту набор простых методов. Это позволяет уменьшить сложность взаимодействия с системой и повысить читаемость кода.
Использование данных паттернов позволяет повысить гибкость архитектуры приложения, уменьшить связанность компонентов и упростить дальнейшее расширение функциональности программы.

Описание выполненных действий
Для выполнения лабораторной работы было разработано консольное приложение, моделирующее работу файлового менеджера.
На первом этапе был реализован паттерн Composite. Для этого был создан абстрактный класс FileSystemItem, который представляет общий элемент файловой системы. От данного класса наследуются два класса: FileItem, представляющий файл, и Folder, представляющий папку. Класс папки содержит список элементов файловой системы и позволяет добавлять или удалять дочерние элементы. Также реализован метод вычисления общего размера папки, который рекурсивно суммирует размеры всех файлов и подпапок.
На втором этапе был реализован паттерн Adapter. Для этого был создан интерфейс IFileSystem, определяющий набор методов для работы с файловой системой: получение списка элементов, чтение файлов, запись данных и удаление элементов. Далее был реализован класс адаптера, который позволяет работать с ранее созданной моделью файловой системы через данный интерфейс.
На третьем этапе был реализован паттерн Facade. Для этого был создан класс SyncFacade, предоставляющий простой интерфейс для выполнения типовых операций, таких как синхронизация папок и создание резервных копий. Данный класс скрывает сложность рекурсивного обхода файловой структуры и взаимодействия с адаптерами различных файловых систем.
В основной программе была продемонстрирована работа всех реализованных паттернов. Была создана иерархия папок и файлов, вычислен общий размер корневой директории, а также выполнена синхронизация данных между различными файловыми системами.

Результат выполненной работы

В результате выполнения лабораторной работы было создано консольное приложение, моделирующее работу файлового менеджера.
Проверка работы программы показала, что: Использование паттерна Composite позволяет удобно представлять иерархическую структуру файловой системы и одинаково работать с файлами и папками. Паттерн Adapter обеспечивает единый интерфейс для работы с различными типами файловых систем, что позволяет использовать одинаковые методы для выполнения операций независимо от типа хранилища. Паттерн Facade упрощает работу с системой, предоставляя клиентскому коду простой интерфейс для выполнения сложных операций, таких как синхронизация данных и резервное копирование. Полученные результаты подтверждают корректность реализации структурных паттернов проектирования и демонстрируют их практическое применение при разработке программных систем.
